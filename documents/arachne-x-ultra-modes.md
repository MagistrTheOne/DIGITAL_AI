# ARACHNE-X-ULTRA — режимы генерации

| | |
|---|---|
| **Компания** | NULLXES LLC |
| **Продукт** | ARACHNE-X |
| **Позиционирование** | Real-time мультимодальная генерация управляемых цифровых людей. |

Внутреннее кодовое имя стека режимов: **ARACHNE-X-ULTRA**. Документ описывает **режимы работы** модели и пайплайна; интеграция с фронтендом для **realtime session** вынесена в отдельный документ: *ARACHNE-X-ULTRA × frontend*.

---

## 1. Режимы по типу входа и выхода

| Режим | Кратко |
|--------|--------|
| **Text-to-Video** | Текстовый промпт → видео цифрового человека. |
| **Image-to-Video** | Опорный кадр / изображение → видео с движением и согласованностью. |
| **Video Continuation** | Продолжение существующего видеоряда по времени (temporal continuation). |
| **Audio-driven avatar generation** | Аудиодорожка (речь) управляет артикуляцией и движением аватара. |
| **Avatar Video Continuation** | Продолжение именно **аватарного** видео с сохранением идентичности и стиля. |

---

## 2. Режимы по инференсу и задержке

| Режим | Кратко |
|--------|--------|
| **Streaming Inference / real-time avatar mode** | Потоковый вывод; минимизация end-to-end latency для живого взаимодействия. |
| **Fast low-latency mode** | Приоритет скорости; допустим компромисс по деталям. |
| **Higher-quality mode** | Приоритет качества; допустима большая задержка и/или выше стоимость вычислений. |
| **Batch / offline mode** | Небоевой режим: пакеты, офлайн-рендер, без жёсткого SLA по latency. |

---

## 3. Режимы по сцене и субъектам

| Режим | Кратко |
|--------|--------|
| **Single-person mode** | Один цифровой человек в кадре / в сессии. |
| **Multi-person mode** | Несколько цифровых людей; разделение идентичностей и взаимодействий. |

---

## 4. Условные режимы (conditioning)

| Режим | Кратко |
|--------|--------|
| **Identity-conditioned mode** | Явная фиксация личности (лицо, стиль, персона) на всём клипе или сессии. |
| **Emotion-conditioned mode** | Управление эмоциональным тоном по меткам или непрерывному сигналу. |
| **Phoneme-conditioned mode** | Синхронизация губ и мимики с фонемной разметкой / ASR-фонемами для точной артикуляции. |

---

## 5. Связь с другими документами

| Тема | Где описано |
|------|-------------|
| **Realtime session mode** + UI/клиент | Отдельный док: **ARACHNE-X-ULTRA × frontend** (не дублировать здесь). |

---

## 6. Примечания для реализации

- Режимы могут **комбинироваться** (например: *Audio-driven* + *Phoneme-conditioned* + *Fast low-latency*), если оркестратор поддерживает матрицу флагов.
- Имена режимов — **контракт** между оркестратором, API и (где нужно) фронтом; при смене enum’ов обновлять док и версию API.
- **ARACHNE DiT 13B** и узлы **Motion Encoder / AV Sync / Muxer** из общей схемы пайплайна остаются общими; конкретный **набор активных узлов** задаётся выбранным режимом и политикой качества.
